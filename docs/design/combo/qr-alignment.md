## 背景

- 你们没有 cc_2025+ 的数据权限,所以 combo 的推理 (predict) 必须在我这边、用最新数据重新跑。
- 数据不开放给你们是有意的: 否则模型选择/调参容易反向过拟合到 2025,样本外评估就失真了。
- 现在每个 combo 提交的形式都不一样 (有的给 config,有的给一大包代码),我每次都要人工辨认,
  想约定一套统一接口。

## 我对 combo 的理解 (请确认对不对)

我把 combo 理解成"一个算法是模型的因子":
- 普通因子: `generate(di)` 里是人写的公式 (如 `rank(close/delay(close,5))`)
- combo: `generate(di)` 里是 `model.predict(X)` —— 把当天的 feature 喂进训练好的模型

也就是说,你们交付的本质是 **训练好的模型 + 推理代码 + feature 清单**,我用它在最新数据上
**重新推理**出预测值,再走 gsim 回测出 pnl。我**不碰**你们怎么训练的。

## 请你们确认的几个问题 (★ 最重要)

### Q1. 推理范式: 是不是"离线训练 + 前馈推理"?

我目前看到的模式是: 离线训练出**按日期命名的 checkpoint** (如每年一个),推理时按"≤当天的最新 checkpoint"
选模型、纯前馈算出当天信号,不在线更新。

- 你们都是这个模式吗?
- 有没有人的 combo 推理时需要**跨日状态** (如 LSTM 的 hidden state 跨天传递)?
- 有没有人是**每天滚动重训** (训练和推理分不开)?

> 这个决定我能不能把推理做成"无状态、按天可重放"的统一形态。

### Q2. 推理一天要多久?

我需要知道单日推理的耗时量级 (毫秒 / 秒 / 更久),以及:
- 必须用 GPU 吗?还是 CPU 能接受 (我这边回测机**没有 GPU**)?
- 全历史 (2020-2025, ~3900 天) 推理一遍大概多久?

> 这决定回测时是"每天实时算"还是"先批量算好缓存成 .npy"。

### Q3. feature 来源

- feature 是不是都来自 cc (`/datasvc/data/cc/...`) 下的字段?
- 有没有用到 cc 之外的、我这边可能没有的数据?
- feature 清单怎么交 (像现在的 `feature_names.csv` 那样列路径,可以吗)?

### Q4. 外部依赖 (优化器等)

有些 combo 的 gsim config 里引用了外部 `.so` (如 `AlphaOpRiskOpt20` 优化器)。
- 这类东西哪些是你们有、我这边没有的?哪些是我有、你们没有的?
- 提交时怎么声明,我才知道要不要我这边补。

### Q5. 实盘衔接

最终这些 combo 要上实盘 (每天推理出当天目标仓位)。
- 实盘那边现在是怎么跑的?是每天跑一次 gsim 出持仓,还是别的机制?
- 理想情况下,回测和实盘应该跑**同一份推理代码**,只换数据和日期。你们认同吗?

## 初步的提交约定 (草案,待 Q1-Q5 确认后定稿)

一个 combo 提交一个目录,包含:

| 内容 | 说明 | 必填 |
|---|---|---|
| 推理代码 | 实现 predict 逻辑的入口 + helper,接受 (data_root, start, end, device, output) 参数 | 是 |
| 模型权重 | 训练好的 checkpoint,**按日期命名** (如 `lgbm_20240102_*.pkl`) | 是 |
| feature 清单 | 用 cc 哪些字段 (路径列表) | 是 |
| gsim config | 回测基准配置 (我可能会改 stats/operations 等参数) | 是 |
| 环境说明 | Python 版本 + pip 依赖及精确版本 | 是 |
| 元信息 | name / version / 提交日期 / 作者 | 是 |

**硬约束** (避免我这边跑不起来):
- 不允许硬编码个人路径 (如 `/home/lhw/miniconda3/...`) —— 路径要么相对,要么从参数传入
- `--device` 要支持 cpu (我这边无 GPU);默认建议自动探测 (有卡用卡,没卡用 cpu)
- 推理代码不能假设数据只到某个固定日期 (我会传 `--end 20251231`)
- config 里不能留待填占位符 (如 `"modify to your XXX.so"`)
- 模型不能在推理时从外部 URL 下载

**不需要你们交**: 预测好的 .npy (我会用你们代码重新生成)、训练代码、数据。
