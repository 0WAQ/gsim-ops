### CRITICAL — 决定 Phase C 能不能切

#### 1. **alpha_src 真实权限模型 vs README 设计意图不一致** 🚨

**实测**(2026-06-04):

```
alpha_src       2750 wbai:alpha-core      ← 但 README 说 root:alpha-core
alpha_pnl       2775 root:alpha-data
alpha_feature   2775 root:alpha-data
alpha_dump      2775 root:alpha-data
alpha_src/<因子>/   2750 wbai:alpha-core  ← 全部 3218 个都是 wbai owner
```

issue:`05-migrate.sh` 跑的是 `chown -R :alpha-core` 只改 group 保留 user,user 那时是 wbai。**README 模型表说 owner 应该是 root**,实际是 wbai。

会话末用户说"忽略我说的,按你看到的来" — 意思是先继续推进,不阻塞。但 Phase C 之前必须决断:
- 选项 A:接受现状,文档更新成 owner=wbai;只允许 wbai-OS-用户跑 ops
- 选项 B:修正成 README 写的 root:alpha-core 2770(group write 位)— 让任何 alpha-core 成员都能 ops
- 选项 C:其他模型

**这事必须先想清楚再做端到端测试**,否则不知道权限错误是 ops bug 还是部署 bug。

#### 2. **端到端 ops 业务还没在 JFS 上跑通**

只跑过单因子 `ops check AlphaWbaiReversal`(2026-06-02 第二轮)。没跑过的:

- `ops submit / resubmit / recheck / rm / pack / approve` 在 JFS 上的完整流水线
- 跨节点并发(2 节点同时 submit / check 同因子)
- factor_lock(per-machine fcntl)+ JFS flock 协同

下一步推荐:**等权限模型决断后,跑 `ops recheck AlphaWbaiReversal -c config.juicefs.yaml`** — 它会:
1. `shutil.move(alpha_src/AlphaWbaiReversal → staging/AlphaWbaiReversal)`
2. `transition: ACTIVE → SUBMITTED`(写 6380)
3. **不会**自动跑 check;再跑 `ops check -c config.juicefs.yaml` 才走 7-stage

风险:动 JFS 上的 AlphaWbaiReversal,但**不动 prod**(prod 在 `/mnt/storage/alphalib/`,独立)。

注意:跑 ops 进程时 supplementary groups 必须含 alpha-core + alpha-data。如果 SSH session 太老,`id -nG` 看不到这俩组,要 SSH 重连或 `sg alpha-core newgrp alpha-data`。

## 给下一个 session 的 ContextNote

- **不要**在 fuse mount 的子目录里跑 unit restart(stale cwd / ESTALE)
- **不要**`systemctl stop redis` — 默认会 SHUTDOWN SAVE 写脏 dump.rdb;用 `redis-cli SHUTDOWN NOSAVE` 或先 `config set save ''`
- **不要**改 redis.conf 加 `appendonly yes` 后 restart — Redis 7 会用空 AOF 优先于 RDB,数据归零;要走运行时 `config set appendonly yes` + `config rewrite`
- redis-py 8.x `from_url()` 不 honor `protocol=2`,直接 `redis.Redis()` 构造
- 144 的 client env 文件叫 `/etc/juicefs/alphalib.env`(不带 -jfs 后缀,join.sh 写的)— 跟 160 master 的 `alphalib-jfs.env` 不要混
- `OPS_<VAR>` env 通过 `Config._resolve_vars` 自动覆盖 `vars:` 块,144 上需要 `OPS_ALPHALIB_ROOT=/storage/vault/alphalib`
- ops 进程的 supplementary groups 必须含 alpha-core + alpha-data,新加组的 SSH session 要重连
